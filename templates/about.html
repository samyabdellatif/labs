Laboratory Schedule Management System

A web application for managing laboratory schedules and lecture bookings across multiple labs.

Technologies Used

  • Backend: Python, Flask, MongoDB
  • Frontend: HTML5, CSS3, JavaScript
  • Database: MongoDB with PyMongo driver

Key Features

  • Multi-lab schedule management (6 laboratories)
  • Weekly timetable with conflict detection
  • User authentication and session management
  • Real-time updates via AJAX
  • Lecture creation and editing

Project Structure

labs/
├── server.py                 # Main Flask application
├── requirements.txt          # Python dependencies
├── .env                     # Environment variables
├── README.md                # Project documentation
└── templates/               # HTML templates
    ├── base.html           # Base template with navigation
    ├── index.html          # Main schedule view
    ├── cpanel.html         # Control panel
    ├── login.html          # User login
    ├── about.html          # About page
    └── _lecture_form.html  # Lecture form partial

Database Collections

  • users: Authentication data (username, password, role)
  • lectures: Schedule data (course, instructor, days, times, lab)
